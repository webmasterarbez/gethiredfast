# Gap Analysis Schema

```yaml
# .course-os/imports/gaps.yaml
coverage:
  well_covered:
    - topic: ""
      sources: []

  partially_covered:
    - topic: ""
      sources: []
      missing: ""

  not_covered:
    - topic: ""
      research_needed: true

research_priorities:
  - priority: 1
    topic: ""
    reason: ""

  - priority: 2
    topic: ""
    reason: ""
```

## Coverage Assessment

- **Well Covered**: Multiple quality sources, comprehensive information
- **Partially Covered**: Some sources but gaps in depth or breadth
- **Not Covered**: No sources, requires web research in Phase 2

## Priority Levels

1. **Critical** - Core topic, must have for course
2. **Important** - Significant topic, needed for completeness
3. **Nice to Have** - Supplementary, can be brief
